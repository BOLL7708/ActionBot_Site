Questions & Answers

Q: What is it?
A: Desbot is a bot for Twitch streamers that can handle chat interactions, rewards and more.

Q: What is the current status?
A: It's currently released as a pre-alpha version, this means major changes will still occur.

Q: Where can I get it?
A:Get the latest release over at github.

Current Form

Desbot started out as a project called "Streaming Widget", as it was a single browser source in OBS that could play back audio and show media on screen.

Over time, the project scope expanded, and it is no longer suitable to be called a widget, so it has now been renamed to a bot. The project is still running as a single browser source but that is also planned to change.

The main features handled by this bot are as follows:

  • Can read Twitch chat, uses this for commands, text to speech through Google, logging and overlays.
  • Can handle Twitch rewards, this means create and update rewards, listen to redemptions and trigger things to happen.
  • Post things to Discord, like chat log, redemptions and screenshots.
  • Can connect to OBS Studio, mainly toggle sources and filters and take screenshots.
  • Can connect to Steam to get currently played game, game meta data and achievements.
  • Can connect to VR accessories like OpenVR2WS, OpenVRNotificationPipe and SuperScreenShotterVR.

Future Plans

There will be a transition from running as a webpage requiring a local webserver with PHP and MySQL to running mainly in Node.JS, this as a lot of the code is the backend that handles all communication. There will still be one or more components that are loaded as browser sources, but those will only be for presentation; to provide overlay elements, video and audio playback.

This transition will also see a move from MySQL as the database to SQLite, this to make this project entirely portable instead of relying on locally installed services. This also includes moving away from PHP as the page templating system and hosting the whole webserver through the same Node.JS instance.

Before all this happens the bot will be worked into a stable state with all the existing features working in the current database, which is the conversion that is taking place right now, moving from hard-coded configuration files to a full on database-connected editor. One thing at a time though.
